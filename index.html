Binary search works by taking a list (in the example above all numbers 0-1000) that are sorted and continually checks the middle of the possibilities, then it shrinks the possibilities by a half, and repeats until the element is found, this allows the item to be found in ln(n) time instead of n time, this is much faster than the default linear search at the cost of needing to sort the array, because most things that are needed are already sorted, binary search can help in almost all searches
